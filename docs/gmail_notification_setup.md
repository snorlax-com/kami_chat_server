# AuraFace 相談メール通知 — Gmail での見分け・フィルタ・通知設定

開発者向けに、**通常相談**と**優先導き**の通知を Gmail 上で分け、見逃しにくくするための手順です。

## メールの違い（一覧）

| 項目 | 通常相談 | 優先導き |
|------|----------|----------|
| `consultationType` | `normal` | `priority_guidance` |
| **差出人（From の表示名）** | `AuraFace｜通常相談` | `至急占い相談｜AuraFace【優先導き】` ※アドレスは `MAIL_FROM` と同じ |
| 件名（例） | `[AuraFace] 新しい相談が届きました` | `【至急占い・緊急】2時間以内要対応｜優先導き｜AuraFace [PRIORITY_GUIDANCE]` |
| フィルタ用（任意） | — | 件名に `至急占い` を含む、または `List-Id` 相当の検索: `list:fortune-urgent.consultations.auraface` |
| 受信一覧の2行目（スニペット） | 「至急ではありません」等（プレヘッダー） | 「2時間以内対応」「至急ご確認」等（プレヘッダー） |
| メールヘッダー | `Importance: normal`, `X-Priority: 3` | `Importance: high`, `X-Priority: 1` など |
| 本文の目印 | 「通常相談」 | 冒頭に「【優先導き】」「2時間以内対応」、識別子 `[PRIORITY_GUIDANCE]` |

**Gmail の「重要」マークについて**: Gmail が自動で付ける「重要」は受信者の利用傾向に依存し、送信側のヘッダーだけでは必ず表示されるわけではありません。至急は **件名先頭を `【至急占い・緊急】`、差出人を `至急占い相談｜…` とし、通常の `[AuraFace]` 始まり件名と一覧上で完全に別パターン**にしています。確実に分けたい場合はフィルタで「重要マークを付ける」を推奨します（下記）。

## 1. Gmail でフィルタを作成する

1. Gmail を開き、上部の**検索ボックス**右の**フィルタアイコン**（スライダー）をクリックします。
2. **件名**欄に次のいずれかを入力します。
   - 優先導きのみ拾う例: `優先導き`
   - より絞る例: `【要確認】【優先導き】`
3. **検索**を押して件数を確認し、問題なければ**フィルタを作成**をクリックします。

### 推奨の検索クエリ例（コピー用）

- `subject:優先導き`
- `"【要確認】【優先導き】"`
- `[PRIORITY_GUIDANCE]`

## 2. フィルタの動作（推奨）

フィルタ作成画面で、次を組み合わせると運用しやすくなります。

- **ラベルを付ける**: 新規ラベル「優先導き」を作成して付与
- **重要マークを付ける**: オン（利用できる場合）
- **迷惑メールにしない**: オン
- 必要に応じて**スターを付ける**: オン

通常相談用に別フィルタを作る場合は、件名に `新しい相談が届きました` を含む、などと指定できます。

## 3. 通常相談と優先導きの見分け方

- **差出人**: 一覧左列で `AuraFace｜通常相談` と `至急占い相談｜AuraFace【優先導き】` が切り替わります（同一アドレスでも表示名が違います）。
- **通常相談の件名**: `[AuraFace] 新しい相談が届きました`
- **優先導きの件名**: `【至急占い・緊急】2時間以内要対応｜優先導き｜AuraFace`（末尾に `[PRIORITY_GUIDANCE]` が付く場合があります）

ロック画面の通知でも、件名に「優先導き」「要確認」が入るため優先度が分かりやすくなります。

### フィルタで「差出人」を条件にする場合

- 優先導き: `from:【優先導き` や `from:AuraFace` と件名 `優先導き` を組み合わせると誤検知が減ります。
- 通常: 件名に `新しい相談が届きました` を含む、かつ差出人表示に `通常相談` を含む、など。

## 4. Android 版 Gmail アプリで通知を強くする（基本）

端末や Gmail のバージョンでメニュー名が多少異なる場合があります。

1. **Gmail** アプリを開く
2. 左上の**メニュー**（三本線）を開く
3. **設定**をタップ
4. 通知を受け取りたい**アカウント**を選択
5. **通知**を開き、メールの受信通知が**すべて**または**高優先度のみ**など、運用に合う設定になっているか確認
6. （利用できる場合）**ラベルの通知**や**ラベルごとの設定**で、「優先導き」ラベルに**通知をオン**にする
7. 端末の**設定 → アプリ → Gmail → 通知**でも、チャネルごとに音や表示を調整できる機種があります

## 5. iPhone 版 Gmail で確認すべき点

1. **設定（iOS）→ 通知 → Gmail** で通知が**許可**されているか
2. **Gmail アプリ**を開き、**設定 → [アカウント] → 通知**で、アプリ内の通知が有効か確認
3. **重要メール**や**優先表示**に関する項目があれば、開発者用受信箱の運用に合わせてオン

※ iOS はシステム側の通知設定が優先されるため、Gmail 単体で「このラベルだけ別音」とまではできない場合があります。その場合はフィルタ＋ラベル＋バッジ表示の組み合わせで運用します。

## 6. Gmail 検索用キーワード

- `subject:優先導き`
- `"【要確認】【優先導き】"`
- `[PRIORITY_GUIDANCE]`
- `subject:"新しい相談が届きました"`（通常相談のみ）

## 7. 見逃し防止の運用

- 優先導きのメールは、**返信・対応が終わるまでアーカイブしない**習慣にする
- **「優先導き」ラベル**を受信後すぐ確認する（フィルタで自動付与）
- アプリ側の**管理者返信ページ**や未対応一覧と併用する

## 関連（サーバー・コード）

- 件名・本文の組み立て: `kami_chat_server/mail/buildConsultationNotification.js`

- 至急の受付方針（デフォルト **24 時間**。任意で JST 時刻帯のみ `kami_chat_server/config/urgentReception.js` ＋環境変数）: `CONSULTATION_URGENT_JST_HOUR_START` / `CONSULTATION_URGENT_JST_HOUR_END`（`kami_chat_server/.env.example` 参照）
- テンプレート: `kami_chat_server/mail/templates/normalConsultation.js` / `priorityGuidance.js`
- テスト: `kami_chat_server` で `npm run test:consultation-mail`
